# ll-todo 开发说明

## 1. 如何分享代码

### 方式 A：通过 GitHub 分享（推荐）
适合多人协作、可追踪历史。

```powershell
git status
git checkout -b feat/你的功能
git add .
git commit -m "feat: 你的变更"
git push -u origin feat/你的功能
```

推送后把仓库链接或 PR 链接发给对方即可。

### 方式 B：源码打包为 zip
适合临时传源码，不依赖 Git 历史。

```powershell
# 在 Compose-ToDo 的上一级目录执行
Compress-Archive -Path .\Compose-ToDo\* -DestinationPath .\Compose-ToDo-source.zip -Force
```

建议先排除大目录（可选）：`.gradle`、`build`、`tmp`。

### 方式 C：只分享本次改动（patch）
适合代码审阅或小范围同步。

```powershell
git format-patch -1
```

接收方可执行：

```powershell
git am xxx.patch
```

## 2. 本地开发环境

- JDK：17
- Android SDK：
  - `platforms;android-35`
  - `build-tools;35.0.0`
  - `platform-tools`
- 构建工具：使用仓库自带 `gradlew` / `gradlew.bat`

Windows 终端建议先设置：

```powershell
$env:JAVA_HOME='C:\Program Files\Java\jdk-17'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
```

## 3. 编译命令

```powershell
git clone https://github.com/wisnukurniawan/Compose-ToDo.git
cd Compose-ToDo
.\gradlew.bat :app:assembleDebug --no-configuration-cache
```

产物路径：

- Debug APK：`app\build\outputs\apk\debug\app-debug.apk`
- Release APK：`app\build\outputs\apk\release\app-release.apk`

## 4. 真机安装与启动

```powershell
adb devices
adb -s <deviceId> install -r -t -g app\build\outputs\apk\debug\app-debug.apk
adb -s <deviceId> shell monkey -p com.lltodo.app.debug -c android.intent.category.LAUNCHER 1
```

安装 Release 包：

```powershell
adb -s <deviceId> install -r -g app\build\outputs\apk\release\app-release.apk
```

## 5. 常见问题

### 1) configuration-cache.lock 被占用

```powershell
.\gradlew.bat --stop
```

然后重新编译。

### 2) 怀疑构建缓存污染（如 NoClassDefFoundError）

```powershell
.\gradlew.bat clean :app:assembleDebug --no-build-cache --rerun-tasks --no-configuration-cache
```

### 3) 手机安装失败
- 确认开启 USB 调试并已授权
- 重启 adb：

```powershell
adb kill-server
adb start-server
```

- 若提示签名冲突，先卸载旧包再安装新包

## 6. 对外分发建议

给其他安卓手机安装，优先使用 Release 包：

1. 构建：`:app:assembleRelease`
2. 分享：`app-release.apk`
3. 附带：版本号、构建时间、变更摘要

若对方安装失败，最常见原因是“手机已装同包名但不同签名版本”，先卸载旧版再安装。
